# 产品需求文档 (PRD) — Excel 小表并大表工具

> 版本: v1.1  
> 日期: 2026-06-07  
> 修订: 补充大表定义、合并写入策略、边界情况、安全措施等

---

## 1. 项目概述

### 1.1 背景
用户在日常数据处理中，需要将多个结构相似的 Excel 小表数据，按列名匹配的方式合并到一张大表中。过程中需要处理列名重复、列名缺失等异常情况。

### 1.2 目标
开发一个轻量级 Python 工具，打包为单文件 exe，用户双击运行，通过交互式对话框完成小表到大表的数据合并。

### 1.3 大表定义
- **大表是列名模板**：仅保留表头行，不含数据行，定义所有合法列名
- **"表名"列是强制列**：大表第 1 列必须为"表名"，小表合并时自动填入来源文件名
- 大表由用户在 Step 1 输入文件名指定，不依赖固定命名

### 1.4 核心原则
- 零第三方依赖（仅使用 Python 标准库）
- 打包为单文件 exe，无需安装 Python 环境
- 交互友好，异常情况弹窗让用户决策

## 3. 功能需求

### 3.1 文件识别与扫描

| ID | 需求描述 | 优先级 |
|----|---------|-------|
| F1 | 程序自动定位自身所在目录作为工作目录 | P0 |
| F2 | 用户通过输入框指定大表文件名，不输入后缀自动补全(.xlsx) | P0 |
| F3 | 目录中除大表外的所有 `.xlsx` 文件自动识别为小表，**排除 `~$` 开头的临时文件** | P0 |
| F3a | **仅识别 `.xlsx` 文件**。`.txt`、`.csv`、`.md`、`.pdf` 等非 xlsx 文件**直接忽略**，不作为小表处理。目录内可安全放置使用说明、README 等引导文件，无需挪动 | P0 |
| F4 | 无小表时提示用户并退出 | P1 |

### 3.2 数据解析

| ID | 需求描述 | 优先级 |
|----|---------|-------|
| F5 | 读取大表的所有 Sheet 名称作为标准 Sheet 集合。小表必须拥有与大表**名称完全一致**的 Sheet 才进行数据合并，Sheet 名称不一致的视为不匹配 | P0 |
| F5a | 小表与大表的 Sheet 名称比对规则：①小表缺少大表某个 Sheet → 跳过该 Sheet，继续处理其他 Sheet；②小表存在大表中没有的 Sheet → 忽略该多余 Sheet；③小表所有 Sheet 都和大表不匹配 → **取消合并该小表**，记录到"取消合并的表名"Sheet，原因标注"Sheet名称不匹配" | P0 |
| F6 | 读取小表和大表的第 1 行作为表头行（列名） | P0 |
| F7 | 使用 Python 内置 `zipfile` + `xml.etree.ElementTree` 解析 xlsx，不依赖 openpyxl 等第三方库 | P0 |

### 3.3 进度反馈

| ID | 需求描述 | 优先级 |
|----|---------|-------|
| F11a | 合并过程中在控制台/日志输出进度（如 `[2/15] 正在处理: 广州.xlsx ...`），防止大量小表合并时用户误以为程序卡死 | P1 |

### 3.4 列名匹配合并

| ID | 需求描述 | 优先级 |
|----|---------|-------|
| F8 | 按列名（表头文字）将小表数据匹配到大表的对应列 | P0 |
| F9 | "表名"列自动填写来源文件名（不含扩展名）。文件名中若含多个 `.`（如 `广州.天河.xlsx`），仅去除最后一个 `.xlsx` 后缀，保留为 `广州.天河` | P0 |
| F10 | 小表列名在大表中不存在时，该列数据丢弃，**并在控制台/日志中输出丢弃的列名** | P0 |

### 3.5 异常处理 — 无列名检测

| ID | 需求描述 | 优先级 |
|----|---------|-------|
| F11 | 扫描小表表头时，检测是否存在**有数据但表头为空**的列 | P0 |
| F12 | 发现无列名列时弹出对话框，显示：文件信息、第几列无列名 | P0 |
| F13 | 对话框提供两个选项：**「忽略该列（记录到异常表）」** / **「取消合并该表」** | P0 |

### 3.6 异常处理 — 同名列值冲突

| ID | 需求描述 | 优先级 |
|----|---------|-------|
| F14 | 小表中出现两列同名列名时（如两个"A"列），在处理该行数据时比对两列值 | P0 |
| F15 | 两值一致 → 自动填入，无需用户干涉 | P0 |
| F16 | 两值不一致 → 弹出对话框，展示：表名、Sheet名、行号、列名、列1的值、列2的值 | P0 |
| F17 | 对话框提供四个选项：**「填入列1的值」** / **「填入列2的值」** / **「忽略（记录到异常表）」** / **「取消合并该表」** | P0 |

### 3.7 对话框行为定义

| 按钮 | 行为 |
|------|------|
| **填入列1的值** | 将列1（首次出现）的值写入大表，继续处理 |
| **填入列2的值** | 将列2（重复出现）的值写入大表，继续处理 |
| **忽略（记录到异常表）** | 该冲突不写入大表，记录到"异常记录"Sheet，继续处理 |
| **取消合并该表** | **立即中止**当前小表的全部合并。由于采用缓存后一次性写入策略，此时该表数据尚未写入结果文件，直接丢弃即可。文件名记录到"取消合并的表名"Sheet，自动开始下一个小表 |

### 3.8 合并写入策略

| ID | 需求描述 | 优先级 |
|----|---------|-------|
| F17a | 每个小表的数据**先缓存到内存**，整表无冲突或用户未取消后，再一次性写入。原因：xlsx 是 zip 格式，无法逐行追加，每次写入都是整文件重建，缓存后一次性写入反而更简单高效 | P0 |

### 3.9 输出文件

| ID | 需求描述 | 优先级 |
|----|---------|-------|
| F18 | 生成新文件 `合并结果.xlsx`，保留原文件不变。若 `合并结果.xlsx` 已存在，弹窗提示"合并结果.xlsx 已存在，是否覆盖？"，用户确认后再覆盖 | P0 |
| F19 | Sheet1: 大表 Sheet1 的表头 + 所有小表 Sheet1 合并后的数据行 | P0 |
| F20 | Sheet2: 大表 Sheet2 的表头 + 所有小表 Sheet2 合并后的数据行 | P0 |
| F21 | 异常记录 Sheet: 记录用户选择"忽略"的所有异常，**纵向排列**（一行一条异常记录） | P0 |
| F22 | 取消合并的表名 Sheet: 记录被取消的表名列表（单列） | P0 |

### 3.10 异常记录表格式

| 文件名 | Sheet | 行号 | 列名 | 列1位置 | 列1值 | 列2位置(若同名) | 列2值(若同名) | 异常类型 | 用户操作 |
|--------|-------|------|------|---------|-------|----------------|---------------|---------|---------|
| 小表.xlsx | Sheet1 | 2 | A | A | 30 | H | 33 | 列名冲突 | 忽略 |
| 小表.xlsx | Sheet1 | - | (无列名) | F | - | - | - | 无列名 | 忽略 |

### 3.11 取消合并的表名 Sheet 格式

| 被取消的表名 | 取消原因 |
|-------------|---------|
| 小标2.xlsx | 用户取消合并 |
| 小标3.xlsx | Sheet名称不匹配 |

> 取消原因取值：`用户取消合并`（用户在异常对话框中选择取消） / `Sheet名称不匹配`（小表所有 Sheet 均与大表不匹配）

## 4. 技术方案

### 4.1 运行环境
- Python 3.6+（开发环境）
- Windows / Linux / macOS（运行环境，exe 在 Windows 上运行）

### 4.2 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| xlsx 解析 | `zipfile` + `xml.etree.ElementTree` | Python 内置，零依赖 |
| xlsx 写入 | `zipfile` + 手动构建 XML | Python 内置，零依赖 |
| GUI 对话框 | `tkinter` | Python 内置，零依赖 |
| 打包工具 | PyInstaller | 唯一构建期依赖 |

### 4.3 文件组成

```
合并脚本.py          ← 单一源文件
合并结果.xlsx        ← 程序输出（运行时生成）
合并日志.txt         ← 运行日志（记录丢弃的列名、进度、异常等）
```

### 4.4 安全措施
- **大表快照保护**：程序启动时将大表完整读入内存，后续合并全部基于内存中的数据。即使大表在合并过程中被外部修改，也不影响本次合并结果。
- 输出文件与源文件分离，源文件全程只读不写。

### 4.5 打包命令

```bash
pip install pyinstaller
pyinstaller --onefile --console 合并脚本.py
```

> 使用 `--console`（而非 `--noconsole`），保留控制台窗口以便用户查看进度和日志输出。

## 5. 边界情况与约束

| # | 场景 | 处理方式 |
|---|------|---------|
| 1 | 目录中没有小表 | 弹窗提示"未找到小表文件"，退出 |
| 2 | 用户输入的大表文件名不存在 | 弹窗提示"未找到指定文件"，重新输入 |
| 3 | 小表没有数据行（仅表头） | 跳过该小表 |
| 4 | 小表有不存在的 Sheet（如只有 Sheet1 没有 Sheet2） | 按 F5a 规则处理：缺少的 Sheet 跳过，全部不匹配则取消合并该表 |
| 4a | 小表所有 Sheet 名称均与大表不匹配 | 弹窗提示"小表 xxx.xlsx 的 Sheet 名称与大表不一致，已跳过"，记录到"取消合并的表名"Sheet，原因"Sheet名称不匹配" |
| 5 | 大表只有表头没有数据行 | 将小表数据逐行追加 |
| 6 | 大表文件被其他程序打开 | 捕获异常，提示用户关闭文件后重试 |
| 7 | xlsx 文件损坏 | 捕获异常，跳过该文件并记录错误 |
| 8 | 合并结果.xlsx 已存在 | 弹窗提示用户确认后覆盖（见 F18） |
| 9 | 目录中存在 `~$` 开头的临时锁文件 | 自动跳过，不识别为小表 |
| 9a | 目录中存在非 xlsx 文件（`.txt`、`.md`、`.csv`、`.pdf` 等） | **自动忽略**，仅对 `.xlsx` 文件进行合并。用户可在目录中放置使用说明等文件 |
| 10 | 大表缺少"表名"列 | 弹窗提示"大表缺少必填列「表名」，请检查大表文件"，退出 |
| 11 | 合并结果.xlsx 被其他程序打开无法写入 | 捕获异常，弹窗提示"无法写入合并结果.xlsx，请关闭文件后重试" |
